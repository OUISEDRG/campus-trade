## 1. 测试计划

### 1.1 测试目标

验证校园二手交易平台所有功能模块的正确性、稳定性、安全性及异常处理能力，确保系统满足毕业设计要求，覆盖全部 15 个功能模块、核心业务流程、边界条件和错误处理场景。

### 1.2 测试范围

| 序号 | 模块 | 测试范围 |
|------|------|----------|
| 1 | 用户管理 | 注册（V1/V2）、登录（BCrypt/MD5双兼容）、信息修改、账号注销 |
| 2 | 商品管理 | 发布、修改、删除、下架、搜索筛选排序分页、轮播推荐、浏览计数 |
| 3 | 订单管理 | 购买、卖家发货、买家收货、取消订单（商品状态恢复） |
| 4 | 砍价功能 | 创建砍价、参与砍价（随机砍价金额）、通过/拒绝、取消、记录查询 |
| 5 | 物品交换 | 发起交换、接受/拒绝、取消、归属变更 |
| 6 | 收藏功能 | 添加收藏、取消收藏、检查状态、收藏列表 |
| 7 | 留言功能 | 添加留言、留言列表查询 |
| 8 | 评价功能 | 添加评价（含评分）、用户评价汇总 |
| 9 | 即时聊天 | WebSocket通信、消息持久化、历史记录、会话列表、未读/已读、在线状态 |
| 10 | 消息通知 | 模板CRUD、发送消息（模板/直接）、未读/已读管理 |
| 11 | 管理员后台 | 用户管理、仪表盘统计、周/月统计、分类统计、Top卖家、砍价管理 |
| 12 | 校区管理 | 校区列表查询 |
| 13 | 文件上传 | 图片上传（大小/格式/魔数校验）、图片访问 |
| 14 | 个人中心 | 统计信息、商品分类列表 |
| 15 | 微信登录 | OAuth回调、微信登录 |

### 1.3 测试环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 10/11 |
| Java | JDK 17 |
| 框架 | Spring Boot 3.2.0 |
| 数据库 | MySQL 8.0 (数据库名: `campus_trade`) |
| 持久层 | MyBatis-Plus 3.5.5 |
| 服务器端口 | 8080 |
| 测试工具 | Postman / Apifox / HTTP Client |
| 浏览器 | Chrome 120+ (开发者工具用于WebSocket测试) |
| 文件存储 | 本地 `{project}/uploads/` 目录 |

### 1.4 测试类型

| 类型 | 说明 |
|------|------|
| 功能测试 | 验证所有API端点的功能正确性 |
| 接口测试 | 验证请求/响应格式、状态码、异常处理 |
| 边界值测试 | 验证输入边界条件和极端值处理 |
| 异常测试 | 验证系统对各种异常场景的处理 |
| 安全测试 | 验证权限控制、输入校验、密码加密 |
| 事务测试 | 验证数据库事务回滚机制 |
